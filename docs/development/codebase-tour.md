# Codebase Tour

[Documentation Home](../README.md)

## Root

The root README explains project intent and links to demos/wiki pages. It also
contains repository-specific notes about covered text behavior, improved DPI
handling, out-of-source building, and Cairo integration.

`buildScripts/` is the operational build entry point. The scripts are not only
helpers; they encode the expected Poppler and FontForge build process.

The root contains `poppler/`, `fontforge/`, and `poppler-data/` trees. These are
used by the local build layout and are not optional for the existing CMake
paths.

## Converter Source

`pdf2htmlEX/src/pdf2htmlEX.cc` starts the program. It owns global `Param param`
and `ArgParser argparser`, registers all CLI options, opens the PDF, and creates
`HTMLRenderer`.

`pdf2htmlEX/src/Param.h` is the shared configuration surface. If an option is
added or changed, start here and in `parse_options`.

`pdf2htmlEX/src/ArgParser.*` is a small typed wrapper around `getopt_long`.
It supports long options, optional short aliases, callbacks, defaults, and two
positional arguments.

## PDF Processing

`pdf2htmlEX/src/Preprocessor.*` is the first pass over the PDF. It is an
`OutputDev` that records used font codes and page extents.

`pdf2htmlEX/src/HTMLRenderer/HTMLRenderer.h` declares the central renderer. It
inherits from Poppler `OutputDev` and overrides state, text, image, path, link,
and page callbacks.

`pdf2htmlEX/src/HTMLRenderer/general.cc` contains the high-level conversion
loop and final manifest assembly.

## Text And CSS

`pdf2htmlEX/src/HTMLRenderer/state.cc` tracks Poppler graphics/text state and
decides when to open a new HTML line, state span, or clip wrapper.

`pdf2htmlEX/src/HTMLRenderer/text.cc` converts Poppler strings to Unicode text
and spacing offsets.

`pdf2htmlEX/src/HTMLTextPage.*` and `HTMLTextLine.*` store text until page end,
then emit absolute-positioned HTML.

`pdf2htmlEX/src/StateManager.h` interns reusable CSS values into compact class
ids. The generated CSS is dumped at the end of conversion.

## Fonts

`pdf2htmlEX/src/HTMLRenderer/font.cc` is one of the highest-risk files. It
bridges Poppler font metadata, embedded font streams, external font lookup,
ToUnicode behavior, glyph widths, FontForge reencoding, web font generation,
and CSS `@font-face` output.

`pdf2htmlEX/src/util/ffw.c` is a C wrapper around FontForge internals. It exists
because FontForge headers and APIs are C-oriented and cannot be included cleanly
from the C++ modules.

## Backgrounds, Images, And Covered Text

`pdf2htmlEX/src/HTMLRenderer/image.cc` does not directly export each PDF image
as an HTML image. It traces image regions for covered-text detection and lets
Poppler's base `OutputDev` behavior proceed.

`pdf2htmlEX/src/BackgroundRenderer/` renders full-page background layers:

- Splash for PNG/JPEG
- Cairo for SVG
- Splash fallback for oversized SVG output when configured

`pdf2htmlEX/src/DrawingTracer.*` uses Cairo to trace char, stroke, fill, image,
clip, and CTM regions. `CoveredTextDetector.*` uses those regions to decide
which text should be visually hidden in the HTML layer and drawn in the
background layer.

## Links, Outlines, And Forms

`HTMLRenderer/link.cc` emits absolute-positioned link rectangles and destination
metadata for JavaScript. Internal GoTo and URI actions are handled. Remote GoTo
and Launch actions are logged as TODO.

`HTMLRenderer/outline.cc` emits nested lists for PDF outline items.

`HTMLRenderer/form.cc` emits simple HTML inputs/divs for text and button form
widgets when `--process-form 1` is enabled.

## Runtime Assets

`pdf2htmlEX/share/manifest` defines the final HTML shell. It includes static
CSS/JS assets, generated CSS, outline HTML, page HTML, and the loading
indicator.

`base.css.in`, `fancy.css.in`, and `pdf2htmlEX.js.in` are CMake-configured into
runtime files. The minified files are generated by `share/build_css.sh` and
`share/build_js.sh`.

## Tests

`pdf2htmlEX/test/test_output.py` is the fastest suite. It verifies that the
converter exits successfully and generates expected files for default, named,
and split-page outputs.

`pdf2htmlEX/test/browser_tests.py` defines visual browser comparisons. It uses
Selenium screenshots and Pillow image diffs. Browser tests require more system
setup and include reference HTML directories under `pdf2htmlEX/test/browser_tests/`.

## C++ And Python Interaction

Python is not part of the converter runtime. Python scripts execute the built
`pdf2htmlEX` binary, prepare test data directories, compare output files, and
support packaging/test helper workflows.
