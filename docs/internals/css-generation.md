# CSS Generation

[Documentation Home](../README.md)

CSS output combines static runtime CSS from `pdf2htmlEX/share/` with
PDF-specific generated CSS from `HTMLRenderer`.

## Static CSS

Static CSS templates:

- `pdf2htmlEX/share/base.css.in`
- `pdf2htmlEX/share/fancy.css.in`

CMake configures them into:

- `pdf2htmlEX/share/base.css`
- `pdf2htmlEX/share/fancy.css`

The minified files are generated by `pdf2htmlEX/share/build_css.sh`:

- `base.min.css`
- `fancy.min.css`

The script uses `3rdparty/yuicompressor/yuicompressor-2.4.8.jar`. If minifying
fails, it copies the uncompressed CSS.

## CSS Class Names

Class names are defined in `pdf2htmlEX/src/util/css_const.h.in` and configured
to `css_const.h`. The names must stay consistent with `base.css.in` and
`pdf2htmlEX.js.in`.

Important class prefixes:

| Prefix | Purpose |
| --- | --- |
| `pf` | page frame |
| `pc` | page content box |
| `pi` | page data for JavaScript |
| `t` | text line |
| `m` | transform matrix |
| `c` | clip box |
| `bi` | partial background image |
| `bf` | full background image |
| `ff` | font family |
| `fs` | font size |
| `fc` | fill color |
| `sc` | stroke color |
| `ls` | letter spacing |
| `ws` | word spacing |
| `_` | whitespace offset |
| `x` | left position |
| `y` | bottom position |
| `w` | width |
| `h` | height |
| `d` | CSS drawing/link rectangle support |
| `l` | link |
| `it` | text input |
| `ir` | radio/button input |

## Generated CSS State Managers

`HTMLRenderer::dump_css` writes CSS from `AllStateManager` in this order:

1. transform matrix
2. vertical align
3. letter space
4. stroke color
5. word space
6. whitespace
7. fill color
8. font size
9. bottom
10. height
11. width
12. left
13. background image size

If `--printing 1`, it also emits a print media block with scaled print values.

## Numeric Value Interning

`StateManager<double, Imp>` stores numeric values in a `std::map` and reuses an
existing class id when the new value is within the manager's epsilon.

Epsilon values are configured in `HTMLRenderer`:

- vertical align: `--veps`
- whitespace: `--heps`
- left: `--heps`
- most other numeric states: `EPS`

This reduces CSS and HTML size but means tiny numeric differences may collapse
to one class.

## Transform CSS

`TransformMatrixManager` writes:

```css
transform:matrix(a,b,c,d,0,0);
```

with `-ms-` and `-webkit-` variants. It ignores translation components because
positioning is represented through separate `left` and `bottom` classes.

PDF and web coordinate systems differ, so matrix terms are sign-adjusted.

## Color CSS

`FillColorManager` writes `color`.

`StrokeColorManager` writes text stroke approximations:

- regular CSS `text-shadow`
- WebKit-specific `-webkit-text-stroke` inside a WebKit media query

Transparent/invalid stroke uses the `_` invalid id class.

## Font CSS

`font.cc` writes font CSS directly to the generated CSS stream:

- `@font-face`
- `.ff<id>` class for remote web fonts
- `.ff<id>` local font-family class for non-embedded external fonts
- hidden default fallback class when a font cannot be processed

## Page CSS

Page frames and content boxes get `w*` and `h*` classes based on page size.

`BGImageSizeManager` can emit background-size rules for page content boxes,
although current Splash/Cairo output mostly emits explicit image elements.

## Embedding Or Linking CSS

The generated PDF-specific CSS stream is written to:

- temp `__css` when `--embed-css 1`
- `dest-dir/css-filename` when `--embed-css 0`

At final assembly, `$css` in `share/manifest` calls `embed_file`, which embeds a
`<style>` block or writes a `<link rel="stylesheet">` depending on
`--embed-css`.
