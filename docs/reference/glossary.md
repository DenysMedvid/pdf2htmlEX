# Glossary

[Documentation Home](../README.md)

`OutputDev`
: Poppler rendering interface. `pdf2htmlEX` implements several `OutputDev`
subclasses to scan PDFs, emit HTML text, and render backgrounds.

`PDFDoc`
: Poppler document object opened from the input PDF.

`GlobalParams`
: Poppler global configuration object. This repository initializes it with the
private poppler-data directory.

`CropBox`
: PDF page box used by default for output when `--use-cropbox 1`.

`MediaBox`
: PDF page box used when `--use-cropbox 0`.

`CTM`
: Current transformation matrix. The converter stores default CTM data for
viewer JavaScript and converts text matrices into CSS transforms.

`Text matrix`
: PDF matrix controlling text placement. Combined with CTM and text state in
`HTMLRenderer::check_state_change`.

`ToUnicode`
: PDF font map from character codes to Unicode. The converter can force it,
ignore it, or use auto mode through `--tounicode`.

`CID font`
: Composite PDF font type with larger code space. The converter allocates
`0x10000` used-code maps for CID fonts.

`Type 3 font`
: PDF font defined by drawing operations. By default rendered into the
background layer; optionally converted through Cairo SVG glyphs and FontForge
when `--process-type3 1`.

`FontForge wrapper`
: `util/ffw.c` and `util/ffw.h`, a C interface for FontForge operations used by
C++ font handling.

`Splash`
: Poppler bitmap rendering backend used here for PNG/JPEG page backgrounds.

`Cairo`
: Graphics library and Poppler backend used here for SVG page backgrounds and
covered-text tracing support.

`Background layer`
: Rendered image or SVG representation of non-text content and fallback text on
a page.

`Text layer`
: Selectable HTML text emitted above the background layer.

`Covered text`
: Text that is fully or partially covered by later non-text drawing. This
repository has custom detection controlled by `--correct-text-visibility`.

`StateManager`
: Template classes that intern CSS state values and assign compact class ids.

`Manifest`
: `share/manifest`, the template used to assemble the final HTML document.

`Data directory`
: Runtime directory containing `manifest`, CSS, JS, icon, and poppler-data.
Configured by `--data-dir`.

`poppler-data`
: CMap and encoding data used by Poppler, installed into the converter data
directory by `installPdf2htmlEX`.

`Split pages`
: Output mode controlled by `--split-pages 1` where each page content is stored
in a separate file and loaded by JavaScript.

`Embedding`
: Storing CSS, fonts, images, JavaScript, or outline data inside the main HTML
instead of writing linked external files.

`Proof mode`
: `--proof` option that draws text in both text and background layers for visual
comparison.

`AppImage`
: Linux application bundle generated by `buildScripts/createAppImage`.

`OCI image`
: Container image generated by build scripts when Docker or Podman is present.
